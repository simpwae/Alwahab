## What this project is

A React + TypeScript storefront generated by Magic Patterns for **Alwahab**, a multi-category online store (electronics, accessories, home & kitchen, gadgets, lifestyle). Currency is **PKR**. Customer side only so far; admin side (Phases 6–10) not started.

Magic Patterns source design: https://www.magicpatterns.com/c/2znc9chrbvqaelzwethnax

## Shared UI primitives (`src/components/ui/`) — reuse these, don't re-create

- **`Button`** — variants: `primary | secondary | ghost | accent | danger-ghost`; sizes: `sm | md | lg`; props `icon`, `fullWidth`. Rounded-xl.
- **`FormField`** — labelled `<input>` with `error`, `hint`, `leadingIcon` props. Handles `aria-invalid` / `aria-describedby`, red error style with `AlertCircleIcon`. **Use this for all Phase 5 forms (login, signup, profile, address) to keep validation styling consistent.**
- **`StatusBadge`** — pill with variants `success | neutral | danger | warning | accent | info`. Critical: it exports the **shared status→color maps** that customer and admin sides must both use:
  - `PRODUCT_STATUS_VARIANT`: Active→success, Draft→neutral, OutOfStock→danger
  - `PAYMENT_STATUS_VARIANT`: Paid→success, AwaitingVerification→warning, Pending→neutral, Failed→danger
  - `FULFILLMENT_STATUS_VARIANT`: Pending→neutral, Confirmed→info, Shipped→warning, Delivered→success, Cancelled→danger
  - **Phase 5 order lists/badges must import these maps** so the account-side badges match Track Order and (future) admin.

## State / contexts

- **`CartContext`** (`context/CartContext.tsx`): `lines`, `isOpen`, `openCart/closeCart`, `addToCart(product, qty)`, `removeFromCart`, `updateQty`, `clearCart`, `subtotal`, `itemCount`. Add-to-cart fires a sonner toast and opens the drawer. Cart is in-memory (resets on refresh).
- **`OrderContext`** (`context/OrderContext.tsx`): `orders`, `addOrder`, `findOrder(orderId, email)` (used by Track Order), `getOrderById(orderId)`. Exposes `generateOrderId()` → `ALW-#####`. Seeded from `data/sampleOrders`.
- **No auth/user context exists yet.** The `User` and `Address` types are defined but unused. Phase 5 will need a user/auth context (mock is fine — prototype only). Wishlist currently is only a local toggle inside `ProductCard`; it is **not** persisted anywhere. Phase 5 should introduce a real wishlist store (likely on the user/account context) so `/account/wishlist` and the header heart share state.
